EventHub

Role Frontend Developer (with @Sanehaakhtar)
Timeline Team project
Year 2025

Overview

EventHub is a Django-based platform where university societies manage events and memberships. Students can join societies, RSVP to events, and participate in discussions.

Frontend development by @AzkaHafeez and @Sanehaakhtar, backend development by @SohSohh.

The Challenge

Societies needed a role-based system that balanced open access for students with structured control for handlers and admins.

We also had to keep membership management, event assets, and feedback in one place without overcomplicating the interface.

The Solution

We built a role-aware dashboard with society management, event creation, and handler applications tied to admin approvals and email notifications.

JWT authentication and a custom user model keep permissions clean across the UI and API.

Tech Stack

Django Django REST Framework SimpleJWT HTML CSS JavaScript SQLite SMTP

Key Features

• User Roles — Viewers (students), handlers (society reps), and admins.

• Society and Membership Management — Join codes, role assignments, and approvals.

• Event Management — Banners, categories, descriptions, and updates.

• Handler Applications — Admin approval workflow with email notifications.

• Comments and Ratings — Event feedback and discussion threads.

• Admin Dashboard — Manage databases and monitor platform stats.
